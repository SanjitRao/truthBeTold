>\n\t\t\n\n\n\n\n\n\n\n\t\t<br>\n\t\t<h2>Intro - Data Visualization Applications with Dash and Python p.1</h2>\n\t\t<br>\n\n\t\t<div class="video-container"><iframe width="560" height="315" src="https://www.youtube.com/embed/J_Cy_QjG6NE" frameborder="0" allow="autoplay; encrypted-media" allowfullscreen></iframe></div><br>\n\t\t\n\n\n\n\n\n\n\t\t<br>\n\n<p>Hello and welcome to an updated series on data visualization in Python. It has been a while since I personally have looked into data visualization in Python, being very familiar and comfortable with Matplotlib. Matplotlib is a fine graphing library, and is the backend to many other packages that allow you to graph, such as Pandas\' <code>.plot()</code> method. While I have been able to make any graph I have ever wanted in Matplotlib, there are a few issues that I have always experienced:</p>\n\n\n\n<ul>\n    <li>Interactivity - Matplotlib graphs can be slightly interacted with, with zooms and such, but any serious interactivity will cost you a lot of time to work in</li>\n    <li>Embeddability - While we can embed matplotlib graphs into other applications, this can *often* be a very tedious process, again, taking up a lot of development time.</li>\n    <li>Aesthetics - While somewhat trivial, it does matter. When the graphs are just for me, it doesn\'t matter much what they look like, but, for presentation, it, again, takes a lot of time to make matplotlib visualizations appealing. If you were trying to sell something based on the graphs, or sell an application with embedded charts, Matplotlib is likely unacceptable. I have personally had clients refuse Matplotlib, not based on the ability to represent data, but because the representation wasn\'t appealing.</li>\n</ul>\n\n<p>For quick graphs to test, prototype...etc, I do still find that I wind up using Matplotlib, mostly because of Pandas\' very nice integration. </p>\n\n<p>That said, *much* of the time, I am not actually just trying to make a stand-alone graph. I am trying to make a graphical user interface (GUI) in some form of fashion, and graphs are just a big part. When looking into the other available charting libraries, Bokeh also stands out significantly, which is easily embeddable as well. </p>\n\n<p>Then I came across Dash, from Plot.ly, which helps you to build web-based data visualization interfaces by tying together Plot.ly, React, and Flask. </p>\n\n<p>Let\'s check it out! I have been using Dash to create a vehicle data monitoring app:</p>\n\n<!--<div class="video-container">\n    <iframe width="853" height="600" src="https://i.imgur.com/pzcv8vy.mp4" frameborder="0" allowfullscreen></iframe>\n</div>-->\n\n<div class="video-container">\n    <iframe width="853" height="600" src="/static/images/dash/interactive-data-viz-GUI.mp4" frameborder="0" allowfullscreen></iframe>\n</div>\n\n\n<p>To create something like this within some sort of desktop GUI framework, using something like PyQt or Tkinter, or via some web app like Flask or Django, and then combining it with any of the interactive JS frameworks, and then incorporating the javascript interactivity with jquery, ajax, and React like we are here...would take you quite a bit of time and code. The app shown above, required less than 100 lines of code for the actual GUI/app itself, many of the lines are actually just for styling purposes to keep the code readable.</p>\n\n<p>To use Dash, we need the following packages: dash, dash-renderer, dash-html-components, dash-core-components, and plotly. These packages also have various dependencies. You can install them with:</p>\n\n<p><code>sudo pip install dash dash-renderer dash-html-components dash-core-components plotly</code></p>\n\n<p>or, on Windows, open cmd.exe as administrator and do</p>\n\n<p>pip install dash dash-renderer dash-html-components dash-core-components plotly</p>\n\n<p>You may have issues with getting MarkupSafe. If so, you can either remedy the compiler issue by following the steps presented <a href="https://stackoverflow.com/questions/34135280/valueerror-unknown-ms-compiler-version-1900" target="blank"><strong>here</strong></a>, or you can just grab a pre-compiled wheel file for markupsafe here: <a href="https://www.lfd.uci.edu/~gohlke/pythonlibs/#markupsafe" target="blank"><strong>https://www.lfd.uci.edu/~gohlke/pythonlibs/#markupsafe</strong></a>. See the second half of <a href="https://pythonprogramming.net/using-pip-install-for-python-modules/" target="blank"><strong>installing packages in Python tutorial</strong></a> for installing .whl packages if you\'re not sure how to install them.</p>\n\n<p>Let\'s show a quick and basic example of Dash in action. This will start off as more lines than what might be required to make just a graph in another framework, but, as we convert it to a fully-fleded GUI, it will be vastly simpler than the others.</p>\n\n<p>To begin, let\'s make some imports:</p>\n\n<pre class="prettyprint lang-py">import dash\nimport dash_core_components as dcc\nimport dash_html_components as html</pre>\n\n<p>Here, we\'re just importing things like the dash lib, various components (things like graph components), and then HTML components (things like div tags...etc). Next, we begin our app:</p>\n\n<pre class="prettyprint lang-py">app = dash.Dash()</pre>\n\n<p>Since Dash is built around the <code>Flask</code> framework, many of these app settings and setups should look familiar if you are familiar with Flask. Next, we create a layout:</p>\n\n<pre class="prettyprint lang-py">app.layout = html.Div(\'Dash Tutorials\')</pre>\n\n<p>In the above case, this would make an app that simply said "Dash Tutorials" on page load. Nothing amazing, but nice and simple! Let\'s run this:</p>\n\n<pre class="prettyprint lang-py">if __name__ == \'__main__\':\n    app.run_server(debug=True)</pre>\n\n<p>Run this, and you should see the following in your console: <code> * Restarting with stat</code></p>\n\n<p>Now, you can go to your browser, heading to the default <a href="127.0.0.1:8050" target="blank"><strong>127.0.0.1:8050</strong></a></p>\n\n<p>You should see something like:</p>\n\n<img class="responsive-img" src="/static/images/dash/basic-dash-layout.png" alt="python dash tutorials" />\n\n<p>Now, this should be fairly easy to understand right now, but pretty quickly we start adding attributes, and embedding child HTML tags inside of here. For the sake of clarity, the actual content of a tag is contained under a parameter called <code>children</code>, which will look like:</p>\n<pre class="prettyprint lang-py">app.layout = html.Div(children=\'Dash Tutorials\')</pre>\n\n\n<p>If you don\'t have embedded tags, or attributes like <code>style</code> or <code>id</code> or <code>className</code> (the Dash equivalent of HTML\'s <code>class</code>), then you might not see the need for explicitly naming the children, but I\'d suggest you do it for any app that is set to grow.</p>\n\n<p>The children can also be a list of items, not just one. Let\'s add a graph, that\'s what we are here for isn\'t it? To do that, we\'ll start with a skeleton:</p>\n\n<pre class="prettyprint lang-py">import dash\nimport dash_core_components as dcc\nimport dash_html_components as html\n\napp = dash.Dash()\n\napp.layout = html.Div(children=[\n    html.H1(children=\'Dash Tutorials\'),\n    dcc.Graph()\n])\n\nif __name__ == \'__main__\':\n    app.run_server(debug=True)</pre>\n\n\n<p>Here, you can see our layout consists of one giant div, which contains the following children: an <code>h1</code> tag of "dash tutorials" and a <code>dcc.Graph</code>. This wont run yet, we need to specify the elements of the graph, so now let\'s build that out. Here\'s a quick graph example:</p>\n\n\n<pre class="prettyprint lang-py">    dcc.Graph(\n        id=\'example\',\n        figure={\n            \'data\': [\n                {\'x\': [1, 2, 3, 4, 5], \'y\': [9, 6, 2, 1, 5], \'type\': \'line\', \'name\': \'Boats\'},\n                {\'x\': [1, 2, 3, 4, 5], \'y\': [8, 7, 2, 7, 3], \'type\': \'bar\', \'name\': \'Cars\'},\n            ],\n            \'layout\': {\n                \'title\': \'Basic Dash Example\'\n            }\n        }\n    )</pre>\n\n\n<p>The ID is required, and we can use this later to manipulate the graph. We then have the <code>figure</code> element, which contains the data and layout for the graph. Here is where we pass all of the data and what type of graph we want, along with other bits of information. Inside of layout, we can add thigns like the title we\'ve added here.</p>\n\n<p>Running this now gives us:</p>\n\n<img class="responsive-img" src="/static/images/dash/basic-dash-graph.png" alt="python dash tutorials" />\n\n<p>I will note that you can change things *live* as your server is running. Every time you save your script, the server should update. Sometimes this doesn\'t work, or you hit an error, and then things can get funky. When all else fails, you can kill the Python running processes to restart things.</p>\n\n<p>Now that we can see how to create graphs, we are going to focus back on interact-ability. Dash gives us the power of the javascript libary React, which we can use to interact in real time with our app, without needing to reload the page. I will exemplify this in the next tutorial.</p>\n\n\n        \n\n\t\t\n\n\t\t\n\t\t\n\n\n\n\t\t<p>The next tutorial: <a title="Interactive User Interface - Data Visualization GUIs with Dash and Python
